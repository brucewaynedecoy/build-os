# Findings Router

`system/workspace/findings/` stores qualified findings promoted from raw run observations.

Create finding artifacts with `buildos-discovery qualify finding`.

## Required Shape

Each finding uses `FIND-NNN/` and contains:

- `finding.md` for the promoted finding record.
- `qualification.md` for the deterministic confirmation test.
- Optional copied confirmation-test and evidence files when generated by the toolkit.

The structured index row lives in [`../../.os/data/findings.jsonl`](../../.os/data/findings.jsonl). The contract is [`../../.os/contracts/finding-contract.md`](../../.os/contracts/finding-contract.md).

## Negative Assertion Pattern

Negative findings qualify only when a repeatable confirmation test asserts the negative condition and passes. A raw observation without a deterministic passing test stays in its source run's `raw-findings.md`.
